# Deployment Guide: IBOR Analyst on Railway.app

## Overview

This guide covers deploying IBOR Analyst to Railway.app with full security guardrails for Phase 1 (closed beta).

**Architecture:**
- PostgreSQL 16 + pgvector (database)
- Spring Boot 3.5.5 (IBOR middleware)
- FastAPI + Python 3.13 (AI gateway with security layer)
- React + Nginx (frontend)

**Deployment Timeline:** ~15 minutes setup + ~5 minutes for builds

---

## Prerequisites

1. **Railway.app account** — https://railway.app (free tier sufficient for MVP)
2. **GitHub repository** — Push code to your GitHub repo
3. **API Keys:**
   - `ANTHROPIC_API_KEY` — Claude API key for LLM
   - (Optional) `OPENAI_API_KEY` — If using embeddings
4. **Email list** — For Phase 1 whitelist

## Phase 1: Setup on Railway.app

### Step 1: Create a New Railway Project

```bash
# Login to Railway CLI (optional, can also do via web)
railway login

# Create new project in a directory
cd ibor-analyst
railway init
```

Or use the web UI: https://railway.app/dashboard → New Project → Deploy from GitHub

### Step 2: Link GitHub Repository

1. Go to https://railway.app/dashboard
2. Click **New Project**
3. Select **Deploy from GitHub**
4. Authorize GitHub and select your `ibor-analyst` repo
5. Railway will automatically detect `railway.yml`

### Step 3: Add PostgreSQL Database

1. In Railway dashboard, click **+ Add Service**
2. Select **PostgreSQL 16**
3. Railway will auto-generate `POSTGRES_PASSWORD` and `DATABASE_URL`

### Step 4: Set Environment Variables

In Railway dashboard, go to **Variables** and add:

**Required Secrets:**
```
ANTHROPIC_API_KEY=sk-ant-xxxxx
POSTGRES_PASSWORD=<generated by Railway>
EMAIL_WHITELIST=user1@example.com,user2@example.com,team@example.com
```

**Security Settings (defaults work for Phase 1):**
```
RATE_LIMIT_ENABLED=true
RATE_LIMIT_RPM=30

EMAIL_WHITELIST_ENABLED=true

MAX_QUESTIONS_PER_DAY=100
MAX_TOKENS_PER_DAY=500000

COST_TRACKING_ENABLED=true
MAX_DAILY_SPEND_USD=50.0

LOG_ALL_REQUESTS=true
ALERT_ON_QUOTA_VIOLATION=true

ENVIRONMENT=beta
VERIFY_SSL=false
```

### Step 5: Deploy

1. Push to GitHub:
   ```bash
   git add .
   git commit -m "Add security layer and Railway deployment config"
   git push origin main
   ```

2. Railway will automatically detect the push and start building:
   - PostgreSQL starts
   - ibor-middleware builds (Maven)
   - ibor-ai-gateway builds (Python)
   - ibor-ui builds (Node.js)

3. Monitor build progress in Railway dashboard

## Security Guardrails (6 Layers)

### Layer 1: Rate Limiting
- **Per IP**: 30 requests/minute (configurable)
- **Header response**: `X-RateLimit-Remaining`, `X-RateLimit-Limit`
- **Enforcement**: 429 Too Many Requests if exceeded

### Layer 2: Input Validation
- **Question length**: 10-2000 characters
- **Banned keywords**: Configurable (SQL injection, XSS patterns blocked)
- **Portfolio code**: Alphanumeric + dash/underscore only
- **Invalid requests**: 400 Bad Request with clear error

### Layer 3: Authentication (Email Whitelist)
- **Phase 1**: Email validation before processing (coming soon)
- **Phase 2**: OAuth with Google/GitHub
- **Phase 3**: API keys + subscription tiers

### Layer 4: Quotas
- **Daily question limit**: 100 questions/day per IP
- **Daily token limit**: 500,000 tokens/day per IP
- **Exceeded**: 429 Too Many Requests with usage stats

### Layer 5: Cost Controls
- **Daily spend limit**: $50/day (Phase 1 default)
- **Tracking**: Real-time token usage and cost estimation
- **Hard stop**: Requests rejected if daily limit exceeded
- **Monitoring**: All spending logged for review

### Layer 6: Monitoring & Logging
- **Request logging**: All requests logged with response time, tokens, cost
- **Quota violations**: Warnings logged and reported
- **Anomaly detection**: Ready for integration with monitoring tools
- **Audit trail**: Complete request history for security review

## Common Issues

### Issue: PostgreSQL connection timeout
**Solution:**
```bash
# Check database service is running
railway logs -s postgres

# Verify DATABASE_URL is set correctly
railway variables list
```

### Issue: FastAPI won't start (dependency error)
**Solution:**
```bash
# Check Python version requirement (3.13+)
# Rebuild from scratch:
railway build --no-cache
```

### Issue: Rate limit too strict
**Adjust in Railway Variables:**
```
RATE_LIMIT_RPM=60  # Increase to 60 requests/minute
```

### Issue: Cost limit hit mid-day
**Investigate:**
```bash
# Check logs for expensive queries
railway logs -s ibor-ai-gateway | grep "cost_usd"

# Temporarily increase limit (emergency only):
railway variables set MAX_DAILY_SPEND_USD=100
```

## Phase 1 to Phase 2 Migration

When ready to expand from closed beta:

1. **Add OAuth:**
   - Create GitHub OAuth app
   - Update `settings.py` to use OAuth middleware
   - Remove email whitelist check

2. **Implement API Keys:**
   - Add user registration endpoint
   - Generate API keys for authenticated users
   - Add API key validation middleware

3. **Adjust Quotas:**
   - Different quotas per user tier
   - Implement subscription/payment integration

4. **Upgrade Monitoring:**
   - Add DataDog/New Relic integration
   - Set up alerts for quota violations
   - Track usage by user/tier
